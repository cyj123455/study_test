# 基于机器学习的农产品价格分析与预测系统 PRD（最终定稿）

## 一、文档概述

### 1.1 文档目的
本 PRD 基于湖南农业大学本科毕业论文开题报告，整合江南果菜市场白菜（矮脚 / 本地 / 小塘）、土豆价格历史数据及对应产区天气数据（含极端天气标识），明确系统核心功能、数据规范、技术边界与验收标准，为毕业设计的系统开发、模型构建及学术验证提供可落地的产品指导，确保功能完全覆盖开题报告中 "数据抓取 - 预处理 - 分析 - 建模 - 可视化" 全研究内容。

### 1.2 核心背景
在农业产业数字化转型背景下，农产品价格受气候、供需、物流等多因素影响波动剧烈，农户、经销商及监管部门面临 "丰产不丰收"、风险规避难、调控缺乏数据支撑等痛点。本系统聚焦白菜、土豆两类核心农产品，融合多源数据与机器学习技术，实现价格分析、短期预测与异常预警，既解决实际市场问题，又丰富农业大数据应用的学术研究场景。

### 1.3 文档范围
- **覆盖数据**：白菜（矮脚 / 本地 / 小塘）、土豆的历史批发价格、产地、销售区域；对应产区天气数据（日平均气温、降水量、日照时长、极端天气标识）；
- **核心功能**：多源数据采集 / 导入、数据预处理、多维度分析、机器学习模型构建与对比、价格预测、异常预警、可视化展示；
- **技术边界**：聚焦 1-7 天短期价格预测，支持 ARIMA、LSTM、SVR、随机森林 4 类算法，评估指标含 MAE、RMSE、MAPE，暂不涉及长期预测（超 30 天）、多品类扩展及邮件 / 短信预警。

## 二、产品定位

### 2.1 核心目标
- **学术目标**：完成毕业设计要求，构建 4 类模型对比的价格预测框架，预测准确率达标（MAE≤0.3 元 / 公斤、MAPE≤8%），形成功能完善的系统原型，支持学术指标与图表导出；
- **应用目标**：为农户提供种植 / 销售决策参考，为经销商优化采购策略，为监管部门提供市场动态监测与异常预警支持。

### 2.2 目标用户
- **核心用户**：毕业设计开发者及指导教师；
- **实际应用用户**：白菜 / 土豆种植农户、农产品批发市场经销商、农业农村局监管人员。

### 4.2 数据质量要求
- **完整性**：白菜、土豆的历史价格数据连续时间跨度≥12 个月，天气数据（含极端天气标识）与价格数据的日期匹配率≥95%；
- **准确性**：价格数据误差≤±5%（与市场实际价格对比），极端天气标识与权威气象记录一致性≥98%；
- **时效性**：爬虫采集频率≥每周 1 次，支持手动触发实时更新。

## 五、非功能需求

### 5.1 性能要求
- 数据采集：白菜 + 土豆 12 个月历史数据抓取≤8 分钟；
- 模型训练：单模型训练（10 万条以内数据）≤30 分钟，4 种模型并行训练≤120 分钟；
- 查询响应：单条件查询≤1 秒，多条件 + 图表渲染≤3 秒，模型预测结果返回≤5 秒。

### 5.2 易用性要求
- 操作门槛：核心流程（数据采集→模型训练→预测展示）操作步骤≤5 步，支持新手引导弹窗；
- 学术适配：支持导出模型评估报告（含 MAE、RMSE、MAPE 指标）、数据分析图表（PNG/PDF 格式），满足毕业论文排版需求；
- 权限区分：学生角色可操作全功能，农户 / 经销商角色隐藏模型训练细节，仅展示预测结果与决策建议。

### 5.3 技术兼容性要求
- 开发框架：前端 Vue 3.x，后端 Python（爬虫 + 模型），数据库 MySQL 8.0+；
- 运行环境：支持 Windows 10+、macOS 12 + 系统，浏览器兼容 Chrome 90+、Edge 90+；
- 可视化组件：集成 ECharts 5.x，支持图表导出为 PNG/PDF 格式。

## 六、验收标准

### 6.1 功能验收
- 数据模块：成功抓取白菜（3 类）+ 土豆价格数据及对应天气数据（含极端天气标识），数据预处理后缺失率≤3%；
- 分析模块：支持时间序列、相关性分析（含极端天气影响），图表展示准确且可交互；
- 模型模块：4 种算法均能正常训练，最优模型短期预测准确率达标（MAE≤0.3 元 / 公斤、MAPE≤8%）；
- 预警模块：能准确识别价格异常，系统内预警响应时间≤1 分钟；
- 交互模块：支持多条件查询、角色权限控制、学术报表导出功能。

### 6.2 学术验收
- 支持导出含 MAE、RMSE、MAPE 的模型对比报告，数据分析图表可直接用于毕业论文；
- 系统功能完全覆盖开题报告研究内容，代码可复用，模型参数可调整，支持学术验证与优化；
- 系统原型符合毕业设计要求，可完成中期检查与答辩展示。
